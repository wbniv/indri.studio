# Make a 404 page

## Context

`wrangler.toml` already declares `not_found_handling = "404-page"` and `docs/DEPLOY.md` confirms unknown paths are meant to serve `/404.html`. Astro generates `dist/404.html` automatically when `src/pages/404.astro` exists — but the page doesn't exist yet, so a bad URL on indri.studio currently falls through to whatever Cloudflare's bare default serves.

The "hook it up" half of the request is already done at the config level. What's missing is the page itself.

Goal: ship a 404 that (a) carries the studio brand without inventing new visual primitives, (b) matches Indri's voice (short, direct, dry — never apologetic-cute or "Oops!"), and (c) gives the visitor an obvious way back.

## Approach

Single file: `src/pages/404.astro`. Use `Base` layout. Reuse the homepage hero's exact typography rhythm — big primary-container word, small on-surface follow-up — so the 404 reads as a first-class studio page rather than a stranded error screen. RingFlare stays on (default) since this is a studio-level page, not an app page; the ambient flares reinforce "you're still on indri.studio".

### Copy

Mirrors the homepage hero structure precisely. The accent-bar sentence echoes the "if it should exist and we'd use it, we build it" tagline — a small wink, not a joke.

```
404
Page not found.

[primary-container left border]
Either we never built it, or the link is stale.
Here's the way back.

[ home ]   [ all apps ]
```

The two CTAs use the existing `.pill-purple` utility for visual consistency with the rest of the studio surface. Anchored at `/` and `/#apps` (the homepage already has `#apps` as a scroll target on the apps section, confirmed in `src/pages/index.astro:41`).

### Visual / layout

- Centered single section, vertically padded like the hero (`min-h-[640px] md:min-h-[760px]`, `py-24 md:py-32`).
- Container: `max-w-[var(--container-max)] mx-auto px-6 md:px-8 lg:px-[var(--spacing-margin)]`.
- Inner content: `max-w-3xl` so the typography breathes the way the hero does.
- `404` rendered with the same scale ramp as `Software` on the homepage: `text-6xl sm:text-7xl md:text-8xl lg:text-[120px] leading-[0.9] tracking-tight font-black uppercase text-primary-container`.
- `Page not found.` rendered with the hero's subhead scale: `text-3xl sm:text-4xl md:text-5xl leading-[1] tracking-tight font-light text-on-surface mt-2`.
- Accent-bar paragraph: `font-body text-body-lg text-on-surface-variant max-w-xl mt-10 border-l-2 border-primary-container pl-6`.
- CTAs: a `flex flex-wrap gap-3 mt-10` row of two `.pill-purple` anchors. Add a hover transition (`transition-colors hover:bg-[color-mix(in_srgb,_var(--color-primary-container)_18%,_transparent)]`) for affordance — `.pill-purple` currently has no hover state, but adding it inline on these two anchors avoids touching `global.css`.

### Metadata

- `<Base title="404" description="Page not found.">` — produces `<title>404 · indri</title>` via Base's existing `fullTitle` logic.

## File to modify

- **New:** `src/pages/404.astro`
- **No changes needed:** `wrangler.toml`, `astro.config.mjs`, `src/layouts/Base.astro`, `src/styles/global.css`.

## Reused primitives

- `src/layouts/Base.astro` — header, footer, RingFlare, ClientRouter, fonts, scroll-shrink.
- `global.css` design tokens — `text-primary-container`, `text-on-surface`, `text-on-surface-variant`, `border-primary-container`, `font-display`, `font-body`, container/spacing vars.
- `.pill-purple` utility (`global.css:301`) for the two return links.
- Hero structure from `src/pages/index.astro:21-38` for the headline/subhead/accent-paragraph rhythm.

## Verification

1. `task build` succeeds and `dist/404.html` exists.
   ```
   ls -la dist/404.html
   ```
2. `task dev` and visit `http://localhost:4321/this-does-not-exist` — Astro's dev server serves `404.astro` for unmatched routes. Confirm:
   - Header (INDRI logo) and footer render persistently.
   - `404` headline is in Phosphor purple at hero scale.
   - "Page not found." subhead renders below it.
   - Accent-bar paragraph renders with the purple left border.
   - Both CTAs (`home`, `all apps`) are visible, clickable, and route correctly (`/` and `/#apps`).
   - RingFlare bursts visible in background over a minute or two.
3. From the dev server, click both CTAs and confirm they land on `/` and on `/` scrolled to the apps section respectively.
4. Production-ish smoke: `task build && pnpm dlx wrangler dev` (or `task deploy` to the workers.dev preview) and visit a bogus path. Confirm Cloudflare serves `/404.html` with a 404 status code:
   ```
   curl -sI https://indri-studio.<acct>.workers.dev/this-does-not-exist | head -n 1
   # expect: HTTP/2 404
   ```
5. Reduced-motion check: in Chrome devtools → Rendering → emulate `prefers-reduced-motion: reduce` and reload. RingFlare should suppress; layout otherwise unchanged.

## Out of scope

- No new components, no new global utilities, no edits to `global.css`.
- No tracking / "report this" UX. The site has no analytics or feedback channel set up yet; adding one belongs in its own plan.
- No per-route 404 variants (e.g., a different 404 inside `/apps/`). Single studio-level page is enough until there's a real reason to split.
