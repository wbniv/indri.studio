# 404 page

> **Status:** shipped + verified in production (2026-05-13). Live at <https://indri.studio/this-does-not-exist> (or any unmatched route). Astro's dev server serves `404.astro` locally at <http://localhost:4321/this-does-not-exist> while `task dev` is running.

## Context

`wrangler.toml` already declares `not_found_handling = "404-page"` and `docs/DEPLOY.md` confirms unknown paths are meant to serve `/404.html`. Astro generates `dist/404.html` automatically when `src/pages/404.astro` exists — but the page didn't exist, so a bad URL was falling through to Cloudflare's bare default.

The "hook it up" half of the request was already done at the config level. What was missing was the page itself.

Goal: ship a 404 that (a) carries the studio brand without inventing new visual primitives, (b) lands a clear-but-deadpan joke in Indri's voice, and (c) gives the visitor an obvious way back. The page should feel like it belongs in the catalogue, not like a stranded error screen.

## The joke

Two parts working together:

1. **The lemur** — a 3D Pixar-style ring-tailed lemur portrait, head-on, staring directly at the camera with a slightly amused smirk. Beady purple pupils (Phosphor accent) inside amber/orange iris haloes, classic black eye-mask, fluffy white face fur. The signature black-and-white ringed tail arches prominently over the head with a solid dark tip. The lemur knows you 404'd.

2. **Accent-bar copy** — inverts the homepage tagline ("If it should exist and we'd use it, we build it.") so the 404 explains itself in the studio's own construction:

   ```
   We build what should exist and we'd use.
   This one didn't make the cut.
   ```

The lemur sells the wink visually (King Julien deadpan); the copy lands it verbally.

## Approach

Single new page, one image asset, no new components. The hand-coded SVG approach (`Lemur.astro` with face + tail paths) was iterated and dropped — flat-vector primitives by hand don't reach Pixar quality, and the user's call was to lean into 3D character art over flat-edge brand consistency for the mascot. The flat-edge `.glass-card` aesthetic still governs UI surfaces; illustration style is a separate layer (see `[[feedback-illustration-style]]` in project memory).

### Layout

Single centered column. Text block at the top of the section, lemur absolutely positioned at the section's bottom edge — its body silhouette sits flush against the footer's top divider. Same layout phone → desktop (no separate stacking rules needed). The split-hero (lemur left, text right) approach in earlier drafts was dropped: anchoring the lemur to the footer makes the wink land harder because the character feels like he's standing *on* the page chrome, not floating beside the text.

- **Section:** `relative overflow-hidden flex-1 min-h-[100svh] md:min-h-[860px] pb-[290px] md:pb-[340px]`. `flex-1` claims all the space between header and footer (requires `<main>` to be a flex column — see Base.astro change below). `pb-[…]` reserves room beneath the text so it can't collide with the absolutely-positioned lemur.
- **Text container:** `text-center pt-12 md:pt-20 lg:pt-24 px-6 md:px-8 w-full max-w-3xl mx-auto`.
- **Headline:** `<h1>` containing two `<span class="block">` lines — `404` at `text-6xl sm:text-7xl md:text-8xl lg:text-[120px]` in `text-primary-container`, then `Page not found.` at `text-3xl sm:text-4xl md:text-5xl` in `text-on-surface`.
- **Accent paragraph:** `max-w-xl mx-auto mt-10 border-l-2 border-primary-container pl-6 text-left` — block is centered under the centered headline, but the text inside stays left-aligned against the purple bar (brand pattern).
- **CTAs:** `flex flex-wrap justify-center gap-3 mt-10` row of `.pill-purple` links with `material-symbols-outlined` icons (`home`, `apps`). Hover bg via inline `color-mix` to avoid touching `global.css`.
- **Lemur:** `absolute bottom-0 inset-x-0 px-6 md:px-8 pointer-events-none` wrapper around `<img>` at `block w-full max-w-[420px] md:max-w-[480px] mx-auto select-none`. `loading="eager"` + `decoding="async"` (hero, no LCP penalty wanted). `pointer-events-none` so the decorative image can't intercept clicks meant for the CTAs above.
- **Idle animation:** `.lemur-idle` keyframes — asymmetric 7s breathe + sway (≤3 px translateY, ≤0.9° rotate, ≤1.4% scale), `transform-origin: 50% 92%` so the upper body wobbles around a planted bottom. Gated on `prefers-reduced-motion: reduce`.

### Image asset

`public/lemur.png` — 1024×684 transparent PNG, generated via ChatGPT (DALL-E / GPT-4o image gen), cropped to bbox and downsized from the 1536×1024 source.

The PNG ships with a transparent background — no alpha-keying needed at build or runtime. Sits cleanly on the page's `#3d3833` surface.

### Metadata

`<Base title="404" description="Page not found." ringFlare={false}>` — produces `<title>404 · indri</title>` via Base's existing `fullTitle` logic. `ringFlare={false}` because the lemur is the focal element and the ambient ring bursts would compete.

## Infrastructure side-shipments

The 404 work also surfaced two reusable bits of plumbing that are landing in the same commit set:

- **`scripts/setup-workers-ai.sh` + `task setup-workers-ai`** — bootstraps `/indri-studio/cloudflare/workers_ai_token` in SSM as a SecureString. Probe-validates the token before pushing. Per-project Cloudflare Workers AI credentials, not shared with `gc-app`.
- **`~/SRC/free-services.md` Recraft entry** — documents Recraft as the only practical free SVG-native generator. Account provisioned for indri-studio (web UI, no API on free tier). Workers AI Flux Schnell and Recraft both evaluated for this lemur job; ChatGPT / DALL-E was picked for character quality (paid, one-off).

## Out of scope

- No edits to `global.css` — all 404 styling lives in `404.astro`.
- No tracking / "report this" UX. The site has no analytics or feedback channel set up yet; that's its own plan.
- No per-route 404 variants (e.g., a distinct lemur on `/apps/`). Single studio-level page until there's a reason to split.
- No on-the-fly image optimization. The PNG is pre-cropped + downsized at check-in time; ~960 KB is acceptable for a 404 hero. If it ever needs to be smaller, run through `astro:assets` or compress with `oxipng`.
